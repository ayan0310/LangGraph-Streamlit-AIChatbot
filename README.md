# LangGraph-Streamlit-AIChatbot
This README covers the implementation of a conversational AI system built using LangGraph, LangChain, and Streamlit. It spans from a basic implementation to a production-ready version featuring SQLite persistence, real-time streaming, and multi-thread conversation management.🤖 LangGraph & Streamlit Chatbot SuiteA comprehensive collection of Python scripts demonstrating how to build a stateful, persistent, and streaming chatbot using the LangGraph framework and Streamlit.🚀 FeaturesState Management: Utilizes LangGraph's StateGraph to manage conversation flow and message history.LLM Integration: Powered by Qwen2.5-7B-Instruct hosted via Hugging Face Endpoints.Persistence: Supports both InMemorySaver for temporary sessions and SqliteSaver for permanent database storage.Streaming UI: Real-time message streaming using Streamlit’s write_stream for a modern "ChatGPT-like" experience.Thread Management: Full support for multiple independent chat threads, allowing users to switch between and resume conversations.📂 Project Structure🧠 Backend (LangGraph Logic)FileDescriptionlanggraph_backend.pyBasic graph setup with InMemorySaver.langgraph_backend_database.pyPersistent graph setup using SqliteSaver and SQLite database.langgraph_backend_streaming.pyBackend configured specifically for streaming output support.🎨 Frontend (Streamlit UI)FileDescriptionstreamlit_frontend.pySimple UI for basic interaction (Non-streaming).streamlit_frontend_streaming.pyUI implementing real-time token streaming.streamlit_frontend_database.pyFull-featured UI with a sidebar for managing multiple persistent threads from the database.streaming_frontend_threading.pySpecialized UI for managing UUID-based threads and history.🛠️ Setup Instructions1. PrerequisitesPython 3.9+A Hugging Face Access Token (with permissions to access the Hub).2. InstallationClone the repository and install the required dependencies:Bashpip install langgraph langchain-huggingface langchain-core streamlit python-dotenv typing-extensions
3. Environment VariablesCreate a .env file in the root directory or ensure your environment has your Hugging Face token:Code snippetHUGGINGFACE_ACCESS_TOKEN=your_token_here
Note: The current scripts contain a hardcoded token for testing; for security, it is highly recommended to move this to your .env file.🎮 How to RunTo run the most advanced version (Persistent with Multi-threading):Bashstreamlit run streamlit_frontend_database.py
Usage Tips:New Chat: Click the "New Chat" button in the sidebar to generate a fresh thread_id.History: Your conversations are saved in chatbot_conversations.db. Previous chats will appear in the sidebar and can be resumed at any time.Streaming: Watch the AI respond in real-time as tokens are generated.🏗️ Architecture DetailThe system follows a standard Stateful Graph pattern:State: A TypedDict containing a list of messages annotated with add_messages to handle history appending.Nodes: An llm_qa function that invokes the Hugging Face model.Edges: A simple linear path from START ➔ llm_qa ➔ END.Checkpointer: The "brain" of the memory, saving the state of the graph at every step to a specific thread_id.
